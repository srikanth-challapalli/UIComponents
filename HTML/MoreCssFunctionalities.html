The styles applied for the above H1 element is
h1{
color: green;
}
h1.withClass{
color:red;
}
h1#withId{
color:blue;
}
h1.anotherClass{
color: blueviolet;
}
For the same same h1 html element we have applied inline style to set the color as orange.
Same h1 html element we have applied color as Green
Same h1 html element we defined class as "withClass" we have applied color as Red
Same h1 html element we defined id as "withId" we have applied color as Blue.

In this case, the priority will always goes in this order
Inline styling > withId styling > withClass styling > h1 element styling.

If you wish to see the color of heading as blue then we have to remove the inline styling. If you wish to see the heading as RED then we have to remove styling with Id. Green if you remove the styling with class.

Let's say if the h1 has two classes assigned "withClass" and "anotherClass" for one I applied Red and for another I applied blueviolet. In this case the one which is defined later in css file will take the priority
if you wish to override this behaviour the we can add "!important" to any of the css styling then only that will take the priority. However, this is not a recommended practice. If you wish to see the RED heading then if we remove the h1 prefix from the styling then we will see RED heading irrecpective of the place in stylesheet. In order to understand why the RED heading is getting displayed we should understand how the specificity will be caliculated in css.

Caliculating Specificity

We the priority in which the styling will take place
Inline styling > id selector > class selector > h1 html element. lets see how the specificity score is caliculated for following styling
h1{
color: green;
}
The above styling has no inline stylying so the score is zero
The above styling has no id selector so the score is zero
The above styling has no class selector so the score is zero
The above styling has only html tag stylying so the score is one
0-0-0-1 = 1 lets see how the specificity score is caliculated for following styling
h1.withClass{
color:red;
}
The above styling has no inline stylying so the score is zero
The above styling has no id selector so the score is zero
The above styling has class selector so the score is one
The above styling has only html tag stylying so the score is one
0-0-1-1 = 11 so this will take priority then previous specificity score.
